# work-bot

`work-bot` - это бот-помощник для автоматизации рабочих процессов вокруг задач и merge request.
Он задуман как расширяемый сервис, который связывает чаты, `Yandex Tracker` и будущие рабочие сценарии в одном месте.

## Текущее состояние

Сейчас в репозитории подготовлен стартовый каркас проекта:

1. базовая структура директорий;
2. шаблон `.env.example`;
3. стартовые JSON-конфиги;
4. минимальный bootstrap приложения;
5. загрузка окружения и конфигов без внешних зависимостей;
6. минимальное runtime-хранилище настроек чатов внутри бота.

## Рабочая модель на старте

На текущем этапе проект исходит из такой модели:

1. права на команды определяются по фактической роли пользователя в `VK Teams`;
2. бот не требует обязательного chat config в проекте;
3. настройки поведения чата должны включаться командами в самом чате;
4. состояние чатов хранится во внутреннем runtime-хранилище бота;
5. на старте достаточно команд только для админов чата и одной базовой автоматизации.

## Быстрый старт

1. Установить `Node.js` версии `20+`.
2. Скопировать `.env.example` в `.env`.
3. Заполнить нужные значения в `.env`.
4. Запустить проект командой `npm start`.

## Что уже умеет каркас

1. Поднимает HTTP-сервер с `GET /health`.
2. Принимает `POST` на путь webhook `VK Teams`.
3. Нормализует входящее сообщение в внутреннюю модель события.
4. Обрабатывает команды `/help`, `/status`, `/enable-review-notifications`, `/disable-review-notifications`.
5. Сохраняет настройки чата во внутреннем runtime-хранилище.
6. Прогоняет ответы через отдельный message service.
7. Принимает webhook `Yandex Tracker` и готовит уведомление о переводе задачи в `Ревью`.
8. Учитывает runtime-маршрутизацию по очередям и доскам для review-уведомлений.
9. Поддерживает базовый runtime-фильтр по полю `Лид разработки`.
10. Позволяет как добавлять, так и удалять runtime-настройки чата командами.
11. Ведет локальный audit log последних событий и показывает краткий след в `/status`.
12. Дает debug-команды `/recent-events` и `/show-chat-settings` для локальной диагностики.

Проверка прав пока временная:

1. клиент `VK Teams` уже выделен в отдельный слой;
2. точные HTTP-вызовы `VK Teams API` пока не подключены;
3. до подключения реального API пользователь считается админом, если его `userId` начинается с `admin`.
